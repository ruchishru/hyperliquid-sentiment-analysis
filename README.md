# Hyperliquid × Fear/Greed Sentiment Analysis

> **How does Bitcoin market sentiment shape trader behaviour and performance on the Hyperliquid DEX?**

A complete end-to-end data science project that joins the Bitcoin Fear/Greed Index with 211,224 Hyperliquid trade records to uncover behavioural patterns, cluster traders into archetypes, build a predictive profitability model, and serve findings through an interactive Streamlit dashboard.

## Project Overview

This project analyses 479 overlapping trading days between the Bitcoin Fear/Greed Index (2018–2025) and Hyperliquid DEX trade data (Jan–Dec 2024) across 32 unique trader accounts.

**What is built:**

| Component | Description |
|---|---|
| `analysis.py` | Full data pipeline: cleaning → feature engineering → 8 analysis charts → predictive model |
| `clustering.py` | KMeans clustering into 4 behavioural archetypes + 4 visualisation charts |
| `dashboard.py` | Streamlit app with 5 interactive tabs covering all findings |
| `notebooks/analysis.ipynb` | Jupyter notebook version of the full analysis |
| `generate_notebook.py` | Script that generates the `.ipynb` programmatically |

## Datasets

### 1. Bitcoin Fear/Greed Index (`fear_greed.csv`)

| Field | Description |
|---|---|
| `timestamp` | Unix timestamp |
| `value` | Numeric index score (0 = Extreme Fear, 100 = Extreme Greed) |
| `classification` | Text label: Extreme Fear / Fear / Neutral / Greed / Extreme Greed |
| `date` | Calendar date (YYYY-MM-DD) |

- **Rows:** 2,644 daily records
- **Coverage:** 2018-02-01 → 2025-05-02
- **Missing values:** None | **Duplicates:** None

### 2. Hyperliquid Trade Records (`hyperliquid_trades.csv`)

| Field | Description |
|---|---|
| `Account` | Trader wallet address |
| `Coin` | Asset traded |
| `Execution Price` | Fill price |
| `Size USD` | Position size in USD |
| `Side` | BUY / SELL |
| `Timestamp IST` | Trade datetime (DD-MM-YYYY HH:MM, IST) |
| `Direction` | Open Long / Open Short / Close Long / Close Short / Buy / Sell / etc. |
| `Closed PnL` | Realised profit/loss on close |
| `Fee` | Trading fee paid |

- **Rows:** 211,224 trade records
- **Coverage:** January 2024 – December 2024
- **Unique traders:** 32 accounts
- **Missing values:** None | **Duplicates:** None

**Date overlap:** 479 days with full Fear/Greed coverage (virtually complete for 2024).

## How to Run

Run all scripts from the **project root** with the virtual environment activated.

### Step 1 — Run the main analysis

```bash
python analysis.py
```

Generates: `data/processed/*.csv`, `charts/chart1–8.png`, `outputs/summary_*.csv`
**Runtime: ~30–60 seconds**

### Step 2 — Run clustering

```bash
python clustering.py
```

Requires Step 1 to be completed first.
Generates: `data/processed/trader_profile_clustered.csv`, `charts/chart9–12.png`
**Runtime: ~15–30 seconds**

### Step 3 — Launch the dashboard (optional)

```bash
streamlit run dashboard.py
```

See full details in the next section.

### Alternative — Jupyter Notebook

```bash
jupyter notebook notebooks/analysis.ipynb
```

Run cells top to bottom. Charts display inline and are saved to `charts/`.

## Methodology

### 1. Data Loading & Documentation
Both datasets loaded with pandas. Zero missing values and zero duplicates in both. Shapes, column types, and date ranges documented.

### 2. Timestamp Parsing & Alignment
- Fear/Greed `date` column: `YYYY-MM-DD` → `pd.to_datetime()` directly
- Hyperliquid `Timestamp IST`: `DD-MM-YYYY HH:MM` → parsed with `format='%d-%m-%Y %H:%M', dayfirst=True`, then `.dt.normalize()` to floor to calendar date
- Inner join on date: **479 overlapping days**

### 3. Sentiment Encoding
- **5-class:** original labels preserved
- **Binary:** Extreme Fear + Fear → `Fear`; Greed + Extreme Greed → `Greed`
- Neutral excluded from binary comparisons for cleaner signal

### 4. Feature Engineering
The `Direction` column (10 distinct values) mapped to flags: `is_long`, `is_short`, `is_close`, `is_win`.

Per-trader daily aggregates (2,340 rows): `daily_pnl`, `trade_count`, `win_rate`, `long_ratio`, `avg_size_usd`, `total_size_usd`, `net_pnl_after_fee`.

Market-wide daily aggregates (479 rows): `total_pnl`, `trade_count`, `unique_traders`, `win_rate`, `long_ratio`, `total_vol_usd`.

All-time trader profiles (32 rows): `total_pnl`, `avg_daily_pnl`, `pnl_std`, `total_trades`, `avg_win_rate`, `avg_size_usd`, `sharpe`.

### 5. Clustering
**Features (9):** core profile features + derived sentiment-sensitivity signals (`pnl_greed_lift`, `direction_shift`, `wr_greed_lift`). Clipped at 2nd/98th percentile, scaled with `StandardScaler`. KMeans k=4 chosen via elbow method analysis. PCA (2 components, 24.1% + 18.7% variance) used for visualisation only.

### 6. Predictive Model
Random Forest (200 trees, depth 8) on 6 features to predict daily trader profitability. 80/20 stratified split with `StandardScaler`. AUC-ROC = **0.81**.

## Strategy Recommendations

**Strategy 1 (Fear):** Cut size ≥30%, halve trade frequency, avoid unconfirmed longs. Sit flat during Extreme Fear.

**Strategy 2 (Greed):** Scale to 1.5× size if win rate >50%, increase frequency, go long. Add short hedges at Extreme Greed.

| F/G Condition | Size | Frequency | Direction |
|---|---|---|---|
| Extreme Fear | Minimal | Very low | Neutral/short |
| Fear | −30% | −50% | No unconfirmed longs |
| Neutral | Normal | Normal | Balanced |
| Greed | +50% (if WR>50%) | Increase | Long-biased |
| Extreme Greed | Reduce longs | Selective | Hedge with shorts |

## Predictive Model Summary

- **Algorithm:** Random Forest, 200 trees, max depth 8
- **AUC-ROC:** 0.81
- **Accuracy:** 76%
- **Profit-day recall:** 98%
- **Loss-day recall:** 40%
- **Top features:** trade count, total volume, position size, F/G score, long ratio, sentiment binary

The Fear/Greed score is a **significant predictive feature**, confirming sentiment provides genuine quantifiable edge beyond behavioural signals alone.
